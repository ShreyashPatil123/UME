# Security Policy

## Supported Versions

Currently, the Unified Memory Engine (UME) is in the Alpha (v3.0.0-alpha) stage. Security updates are applied directly to the `main` branch. 

| Version | Supported          |
| ------- | ------------------ |
| 3.0.x   | :white_check_mark: |
| < 3.0   | :x:                |

## Reporting a Vulnerability

We take the security of UME very seriously. If you believe you have found a security vulnerability in UME, please report it to us confidentially.

**Do not report security vulnerabilities through public GitHub issues.**

Instead, please email the core maintainers directly at: **security@ume-project.org**.

Please include the following information in your report:
*   A description of the vulnerability and its potential impact.
*   A detailed description of the steps required to reproduce the vulnerability (including proof-of-concept scripts or code if available).
*   Your system information (OS, compiler, UME version/commit hash).

You should expect a response within 48 hours acknowledging receipt of your report. We will then keep you informed of our progress as we investigate and remediate the issue. We aim to fix critical vulnerabilities as swiftly as possible and will coordinate a public disclosure with you once the patch is released.
